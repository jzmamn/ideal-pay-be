# ideal-pay-be — Spring Boot Backend

## Stack
- **Java 21**, **Spring Boot 4.0.6**, **Gradle** (Groovy DSL)
- **MySQL 8** via Spring Data JPA + Hibernate
- **Spring Security** + **JWT** (JJWT 0.12.6) — stateless auth
- **MapStruct 1.6.3** — DTO ↔ Entity mapping
- **Lombok** — boilerplate reduction
- **MVEL2** — MVEL formula evaluation (payroll calculations)
- **iText html2pdf** — PDF payslip generation
- **Apache POI** + **OpenCSV** — XLSX/CSV import & export
- **Spring Mail** — email dispatch

## Root Package
`com.payroll`

## Package Structure
```
com.payroll
├── config/          # Spring configs (Security, Password encoder, CORS, etc.)
├── controller/      # @RestController — thin, delegates to service
├── converter/       # JPA AttributeConverters (Boolean ↔ Y/N, Boolean ↔ 0/1)
├── dto/
│   ├── request/     # *RequestDTO — inbound payloads
│   └── response/    # *ResponseDTO — outbound payloads
├── entity/          # @Entity JPA classes
├── exception/       # Custom exceptions (ResourceNotFoundException, FormulaEvaluationException, …)
├── mapper/          # MapStruct interfaces (*Mapper)
├── repository/      # Spring Data JPA interfaces (*Repository)
└── service/         # Service interfaces + Impl classes
```

## Key Conventions

### Controllers
- All routes prefixed `/payroll/<resource>` (e.g. `/payroll/branch`)
- Always return `ResponseEntity<ApiResponseDTO<T>>`
- Use `@Valid` on request body params
- `@RequiredArgsConstructor` for injection via Lombok
- Standard CRUD: GET list, GET by id, POST (201), PUT, DELETE

### ApiResponseDTO
Wrapper for all responses — use the static factory methods:
```java
ApiResponseDTO.success("Message", data)   // success: true
ApiResponseDTO.error("Message")           // success: false
```

### Services
- Define a **Java interface** (e.g. `BranchService`), implement separately (e.g. `BranchServiceImpl`)
- Service impl annotated with `@Service`, `@RequiredArgsConstructor`

### Entities
- All auditable entities carry: `createdBy` (FK → `Usr`), `createdDate`, `modifiedBy`, `modifiedDate`
- Boolean `isActive` stored as `CHAR(1) 'Y'/'N'` via `BooleanToYNConverter`
- `code` column is unique, auto-generated by the backend (not from user input)
- `id` is `BIGINT AUTO_INCREMENT`
- Every table has a **default row** at `id = -1` / `code = 'DEFAULT'` (system sentinel, never deleted)

### Mappers (MapStruct)
- `componentModel = "spring"`, `nullValuePropertyMappingStrategy = IGNORE`
- `toEntity(RequestDTO)` — always ignore `id`, `code`, `isActive`, audit fields
- `toResponseDTO(Entity)` — flatten audit user references to `*Id`, `*Code`, `*UserName` fields
- `updateEntityFromDTO(RequestDTO, @MappingTarget Entity)` — patch style, ignores system fields

### DTOs
- Request DTOs: `*RequestDTO` — validated with Jakarta Bean Validation
- Response DTOs: `*ResponseDTO` — include flattened audit user info

### Converters
- `BooleanToYNConverter` — `'Y'`/`'N'` CHAR column
- `BooleanToZeroOneConverter` — `1`/`0` INT column

### Profiles
| Profile | Usage |
|---------|-------|
| `dev`   | Local MySQL (`localhost:3306/ideal_pay`, user `root`/`1234`), DDL none, verbose logging |
| `qa`    | QA environment |
| `prod`  | Production |
| `demo`  | Demo environment |

Active profile set in `application.properties` (`spring.profiles.active=dev`).

### Database
- DB name: `ideal_pay`
- Schema managed via SQL dump files in `ideal-pay-db` repo (not Flyway)
- `spring.jpa.hibernate.ddl-auto=none` — never let Hibernate modify the schema

### Security
- JWT access tokens expire in 15 min (`jwt.access-token-expiry-ms=900000`)
- Refresh tokens valid for 7 days
- Override `JWT_SECRET` env var in production

### Exception Handling
- `ResourceNotFoundException` — throw when entity not found by id; results in 404
- `FormulaEvaluationException` — thrown when MVEL formula fails evaluation
- Global handler maps these to `ApiResponseDTO.error(...)` responses

## Build & Run
```bash
./gradlew bootRun                          # dev profile (default)
./gradlew bootRun --args='--spring.profiles.active=qa'
./gradlew build
```
